## PRQ1
Prompt:
Erstelle mir einen kurzen GitHub-Workflow für PRQ1: neue Datei anlegen, Branch, Commit, Push, Pull Request, Merge. Nur stichwortartig.

KI-Antwort:
- Neuer Branch von main erstellen (prq1-add-file1)
- File1.txt anlegen und Inhalt einfügen
- git add .
- git commit -m "prq1: add File1.txt"
- git push origin prq1-add-file1
- Pull Request auf GitHub erstellen
- Merge durchführen

Reflexion:
Ich habe die Branch-, Commit- und Push-Schritte übernommen.

## PRQ2
Prompt:
Erstelle mir einen kurzen GitHub-Workflow für PRQ2: bestehende Datei ändern, Branch, Commit, Push, Pull Request, Merge. Nur stichwortartig.

KI-Antwort:
- main auschecken und aktualisieren (git checkout main / git pull)
- Neuer Branch erstellen (prq2-edit-file2)
- File2.txt bearbeiten und Zeile hinzufügen
- git add .
- git commit -m "prq2: edit File2.txt"
- git push origin prq2-edit-file2
- Pull Request erstellen
- Merge durchführen

Reflexion:
Ich habe alles gleich umgesetzt.

## PRQ3
Prompt:
Erstelle mir einen kurzen GitHub-Workflow für PRQ3: Datei löschen, Branch, Commit, Push, Pull Request, Merge. Nur stichwortartig.

KI-Antwort:
- main auschecken und aktualisieren
- Neuer Branch erstellen (prq3-delete-file3)
- Datei löschen (git rm File3.txt)
- git commit -m "prq3: delete File3.txt"
- git push origin prq3-delete-file3
- Pull Request erstellen
- Merge durchführen

Reflexion:
Mit KI ist es viel einfacher und schneller wenn man was erledigen will.
